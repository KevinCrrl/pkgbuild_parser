
# Contributing to pkgbuild parser

You can contribute to PKGBUILD-Parser by following these guidelines and recommendations:

## How to contribute???

All type of contributions are welcome, some examples of contributions:

- Report bugs
- Requests and send new features
- Help reviewing PRs and solving issues
- Improve the code and the docs
- Help with translations (English and Spanish) (You can use a web translator)

## Pull Request Requirements

All the PRs must follow these requirements:

- Comply with PEP-8 throughout the Python code.
- Use flake8 to analyze your code (In GitHub this is automated via Actions)
- Test your code (before and after) using a real PKGBUILD
- Don't add new dependencies, open a issue or discussion if you consider necesary a new dependency
- Document your changes in the PR (and in the code if necesary), it is used for the CHANGELOG

### AI Usage Policy

Because this project is also available in Codeberg, this project respects the Codeberg's terms of use, and the point number 7 in the [Allowed Content & Usage (Section 2)](https://codeberg.org/Codeberg/org/src/branch/main/TermsOfUse.md#2-allowed-content-usage) in these terms of use, the *projects that mostly consist of code written by "generative AI"-tools* must not be shared.

Therefore, I decided not to accept the PRs and issues generated by AI, if you are contributing via GitHub, the use of AI is the responsibility of EACH contributor, including the responsibility to audit, correct, and to create the PR or issue themselves without automation that could be interpreted as SPAM or lead to misunderstandings despite their good intentions as a contributor. My sole responsibility in this case will be to review, correct if necessary, and accept or reject the contribution in the event that there is excessive use of AI that is counterproductive to the project’s progress.
